rigctl, rot ctl, rigmem, rigsmtr, rigswr API

You may use the following API demo methods:
curl localhost/rigctl_api/trx/0/frequency
curl -XPOST localhost/rigctl_api/trx/0/frequency -d'{"newValue":"145425000"}'
curl localhost/rigctl_api/trx/0/mode
curl -XPOST localhost/rigctl_api/trx/0/mode -d'{"mode":"FM","passband":9000}'
curl -XPOST localhost/rigctl_api/trx/0/mode -d'{"mode":"USB","passband":2400}'
curl localhost/rigctl_api/trx/0/split_frequency curl -XPOST localhost/rigctl_api/trx/0/split_frequency -d'{"newValue":145725100}'
curl localhost/rigctl_api/trx/0/split_mode curl -XPOST localhost/rigctl_api/trx/0/split_mode -d'{"mode":"FM","passband":9000}' curl localhost/rigctl_api/trx/0/split_frequency_mode curl -XPOST localhost/rigctl_api/trx/0/split_frequency_mode -d'{"frequency":145425300, "mode":"FM","passband":9000}' curl localhost/rigctl_api/trx/0/split_vfo curl -XPOST localhost/rigctl_api/trx/0/split_vfo -d'{"split_mode":1,"tx_vfo":"VFOA"}' curl -XPOST localhost/rigctl_api/trx/0/split_vfo -d'{"split_mode":0,"tx_vfo":"VFOA"}' curl localhost/rigctl_api/trx/0/tuning_step curl -XPOST localhost/rigctl_api/trx/0/tuning_step -d'{"newValue":5000}' curl -XPOST localhost/rigctl_api/trx/0/tuning_step -d'{"newValue":10000}' curl localhost/rigctl_api/trx/0/level/SQL curl -XPOST localhost/rigctl_api/trx/0/level/SQL -d'{"newValue":1}'

Have a look at the rigctl_api git repository for source code and documentation!
